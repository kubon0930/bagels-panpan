# Bagels Panpan. 公式サイト 運用マニュアル（店舗向け）

このマニュアルは、Bagels Panpan. 公式サイトを **日々運用する方（店舗スタッフ）向け** のガイドです。
プログラミングの知識は不要です。エンジニア向けの技術情報は `README.md` にあります。

---

## 1. サイトの全体像

| ページ | URL | 内容 |
| --- | --- | --- |
| トップページ | `/` | お店の紹介（Hero・Concept・その日並ぶベーグル・こだわり・アクセス・FAQ） |
| 予約販売ページ | `/reserve` | お客様がベーグルを事前予約するページ |
| 管理画面 | `/admin` | 店舗スタッフ専用。ラインナップ・販売日・商品・注文の管理 |
| 法務ページ | `/legal/…` | 特商法表記・プライバシーポリシー・キャンセルポリシー |

**設計の基本方針**

- 最新情報（営業日・臨時休業・焼き上がり）は **Instagramに集約** し、サイト各所からInstagramへ誘導しています
- 予約導線は「予約販売を見る」ボタンに統一されています（スマホでは画面下部に常に表示）
- サイトの内容は Instagram と管理画面で運用でき、**日常運用でコードを触る必要はありません**

---

## 2. 日々の運用ですること・しなくていいこと

### すること
- **Instagram の更新**（営業日・オープン時間・臨時休業・焼き上がり情報）→ サイトは自動的にInstagramへ誘導します
- **予約販売をする週**：管理画面で販売日と商品を登録して公開（→ 4章）
- **予約が入ったら**：管理画面で注文を確認、当日はCSVで取り置きリストを出力（→ 5章）
- **ラインナップが変わったら**：管理画面「ラインナップ」で商品を編集（→ 3章）

### しなくていいこと
- サイトの文章・デザインの修正（必要な場合は開発者に依頼 → 7章）
- サーバーの管理・バックアップ（Vercel / Supabase が自動で行います）

## 3. ホームページのラインナップ管理（/admin/lineup）

トップページ「その日並ぶベーグル」に表示される商品を編集できます。
**変更は公式サイトに最大1分ほどで自動反映されます。**

### 管理画面に入る
1. `https://（サイトURL）/admin/login` を開く
2. 登録済みの管理者メールアドレスとパスワードでログイン
3. メニューから「ラインナップ」を選ぶ

### 商品を追加する
1. ページ下部の「ベーグルを追加」で商品名（英字）と日本語名を入力
2. **イラスト（ベース色 × トッピング）** を選ぶ
   - ベース色（9種）：プレーン / こんがり / 全粒粉 / チーズ / チョコ / 抹茶 / ベリー / パンプキン / 季節限定
   - トッピング（8種）：なし / セサミ / 黒ごま / 焼きチーズ / チョコチップ / シナモン / シュガー / シード
   - 選ぶとすぐ右のプレビューに反映されます（例：抹茶 × 黒ごま、ベリー × シュガー）
3. 「追加」を押す

### 商品を編集する
各商品カードで、次をその場で変更できます（入力欄からフォーカスを外すと自動保存）。

- 商品名 / 日本語名 / 説明文 / タグ（例：Limited）
- **商品写真**：「画像を選ぶ」からスマホの写真をアップロード（5MBまで）。写真があれば写真、なければイラストが表示されます
- **イラスト**：ベース色とトッピングをピルで選択
- 表示順（数字が小さいほど上・左に表示）
- **表示中 / 非表示**：非表示にするとサイトから消えます（データは残ります）

## 4. 予約販売の運用（販売日 → 商品 → 公開）

予約販売は「販売日」単位で運用します。流れは3ステップです。

### STEP 1：販売日を作る（/admin/sales-days）
1. 「＋新しい販売日」で日付を作成（最初は下書き＝非公開）
2. **受け取り時間帯** を追加（例：12:00〜12:30、上限10件）
3. **受付開始／終了日時** を設定（この期間だけお客様が予約できます）

### STEP 2：商品を登録する（/admin/items）
1. 販売日を選ぶ
2. **定番から追加**：「定番に登録」済みの商品はワンタップで追加できます（価格は登録時のもの・販売数10で入るので、あとから調整）
3. 新しい商品は下のフォームから商品名・価格・**販売数（在庫）** を入力して追加
4. 各商品で季節限定/おすすめ・アレルギーメモ・商品画像を設定。**「イラストを変更」を開くと**、写真がないときに予約ページへ表示されるベーグルイラスト（ベース色×トッピング）を選べます
5. 毎回使う商品は「**定番に登録**」を ON にしておくと、次回から「定番から追加」に並びます

### STEP 3：公開する
販売日カードの「**公開**」「**受付中**」を ON にすると、お客様の `/reserve` に表示され予約できるようになります。

### 受付を止めたいとき
- 「受付中」を OFF にする（ページには表示されるが「受付終了」表示になる）
- または「公開」を OFF にする（ページから消える）

### お客様側の流れ（参考）
`/reserve` で商品と数量を選ぶ → 受け取り時間・名前・電話・メールを入力 → 予約確定。
予約番号（**BP-年月-0001** 形式）が発行され、完了画面とメールに表示されます。
在庫はシステムが自動管理し、同時に予約が入っても販売数を超えることはありません。売り切れは自動で「売り切れ」表示になります。

### 支払いについて
- 現在の設定：**予約のみ・店頭でお支払い**（現金・PayPay）
- 将来オンライン決済（Stripe）に切り替え可能です（開発者に依頼。設定手順はREADME参照）

## 5. 注文の確認・当日の準備（/admin/orders）

- 注文一覧を **販売日・ステータス・支払い・名前/商品** で絞り込みできます
- 注文をタップすると詳細画面で以下ができます
  - ステータス変更（受け取り済みなど）
  - **キャンセル**（在庫が自動で戻ります）
  - 管理メモの入力
- **CSVで出力**：当日の取り置き準備・電話確認用の一覧をダウンロード（Excelでそのまま開けます）

## 6. 画像の差し替え

`public/images/` フォルダの画像を差し替えて GitHub に push すると、自動で反映されます（開発者作業）。

| ファイル | 用途 |
| --- | --- |
| `panpan-logo-circle.jpg` | 丸いネイビーロゴ（ヘッダー・フッター・ベーグルの穴の中 など） |
| `panpan-storefront.jpg` | 店舗外観写真。**置くだけで** PCトップのメインビジュアルと Access の写真に自動で切り替わります（未設置の間はイラスト表示） |
| `ogp-placeholder.jpg` | SNSでシェアされたときの画像（1200×630px） |

※ ラインナップ・予約商品の写真は管理画面からアップロードできるので、開発者作業は不要です。

## 7. 文章・営業情報を変えたいとき（開発者に依頼）

以下は `data/site.ts` という1つのファイルにまとまっており、開発者が数分で変更できます。

- 営業日・営業時間・定休日（例：水金土 → 変更）
- 住所・アクセス・支払い方法
- FAQ の質問と回答
- Instagram・食べログのURL
- Hero やコンセプトの文言

依頼するときは「**data/site.ts の◯◯を◯◯に変えてほしい**」と伝えるとスムーズです。

## 8. よくある質問（運用Q&A）

**Q. 臨時休業になった**
→ Instagramで告知すればOK。サイトは各所で「最新情報はInstagramへ」と案内しています。予約販売中の日なら、管理画面で該当販売日の「受付中」をOFFに。

**Q. 予約ページが「準備中です」と表示される**
→ 公開中の販売日がない状態です。販売日の「公開」がONか、受付期間内かを確認してください。

**Q. 管理画面の変更がサイトに出ない**
→ トップページは最大1分の時間差があります。1分待ってブラウザを再読み込みしてください。

**Q. 管理画面に入れない**
→ ログインメールが管理者登録（`app_admins`）されているか、パスワードが正しいかを確認。不明な場合は開発者へ。

**Q. お客様から予約キャンセルの連絡が来た**
→ 管理画面 → 注文詳細 → キャンセル。在庫は自動で戻ります。

**Q. 予約商品の写真がアップロードできない**
→ 5MB以下のJPG/PNGかを確認。それでも失敗する場合は開発者へ（Supabaseの設定確認）。

## 9. 仕組み（ざっくり技術構成）

| 役割 | サービス |
| --- | --- |
| サイト本体 | Next.js（Vercel でホスティング。GitHub の main に push すると自動デプロイ） |
| 予約・商品・注文データ | Supabase（データベース。管理画面のログインもここ） |
| オンライン決済（未使用・切替可能） | Stripe |
| 予約完了メール（任意設定） | Resend |
| 最新情報の発信 | Instagram（@bakery_panpan） |

- データベースの変更履歴は `supabase/migrations/` に保存されています（0001〜0004 適用済み）
- 環境変数（APIキー等）は Vercel に設定されています。**service_role キーは絶対に公開しないでください**
- 独自ドメインを取得したら：Vercel にドメインを追加 + `data/site.ts` の `siteUrl` を変更（開発者作業）
